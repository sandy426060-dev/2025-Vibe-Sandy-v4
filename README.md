# 🍢 2025 Vibe Coding 期末專案：走進夜市就回不了頭的台灣小吃清單

## 🌟 1. 專題願景 (Vibe)
* **目標對象：** 來台灣旅遊的外國人，或是半夜肚子餓不知道要吃什麼的大學生。
* **設計風格：** **現代夜市風 (Modern Night Market)**。捨棄傳統過於雜亂的霓虹色，改用深鐵灰色 (#2b2b2b) 背景營造夜晚的沈浸感，搭配如炸物般的金黃色 (#ffaa00) 作為視覺亮點，呈現「越夜越美味」的高級質感。
* **核心功能：** 1. 精選 13 道台灣夜市經典美食，分為「鹹食飽胃區」與「甜點飲料區」。
    2. 每張卡片皆附有「👀 哪裡吃？」按鈕，點擊可直接 Google 搜尋該美食。
    3. 實作「回到頂部 (Back to Top)」按鈕，優化手機瀏覽體驗。

## 🤖 2. Gemini 協作紀錄 (Prompt Log)
*我餵給 Gemini 最關鍵的一個 Prompt：*
> 「請幫我把圖片換成對應的食物插圖（如果有辦法的話）。然後順序幫我轉換一下，鹹甜分開不要穿插，『台灣夜市必吃美食懶人包』要有一個吸引人的大標題（例如：走進夜市就回不了頭的台灣小吃清單）和一張令人垂涎的背景圖。」

### 🔄 迭代優化過程 (Iterative Phase)

**Round 1：基礎優化**
* **問題：** 初版網頁背景太暗看不清楚，按鈕沒有實際功能，且美食項目不符合需求。
* **修正：** 調整背景亮度提升閱讀性，實作 Google 搜尋按鈕功能，並置入指定的 13 道台灣小吃資料。

**Round 2：結構與氛圍**
* **問題：** 鹹食與甜食混雜在一起，且首頁缺乏夜市的熱鬧氛圍。
* **修正：** 將版面區分為「鹹食區」與「甜點區」，並重新設計 Hero Header，加入夜市背景圖與吸睛標題。

**Round 3：內容擴充與除錯 (最關鍵的一步)**
* **問題：** 甜點選項太少，且部分圖片連結不穩定，導致預覽時出現破圖。
* **修正：** 新增紅豆湯、豆花等 4 樣甜點。針對破圖問題，在嘗試 AI 建議的圖源後，我決定手動尋找並替換成更穩定的圖片連結，確保網頁呈現完美效果。
  
*在開發過程中，你遇到了什麼 Bug？Gemini 怎麼幫你修好的？*
* **遇到的問題：** 1. **圖片顯示異常**：一開始 AI 使用隨機圖庫，導致圖片與食物對不上（例如雞排變成風景照）。
    2. **破圖問題**：在 iPad 上預覽時，維基百科的外部連結圖片無法顯示，導致畫面空白。
* **Gemini 的建議與我的解決方案：** * Gemini 建議使用 `wsrv.nl` 圖片代理服務來修復連結，並嘗試更換圖源。
    * **最終解決方案（由我執行）：** 我決定自己上網尋找穩定且正確的圖片網址，並手動進入 HTML 程式碼中，將連結替換成我挑選過的照片。這不僅解決了破圖問題，也確保每一張照片看起來都最好吃。

## 📂 3. 檔案結構說明
* `index.html`：專案主體，採用 **Single File Component** 架構，將 HTML 結構、CSS 樣式設計與 JavaScript 互動邏輯全部整合在單一檔案中，方便部署與維護。

## 📈 4. 自我心得與反思
* **心得：** 這是第一次體驗 Vibe Coding，我發現最有趣的不是寫出完美的程式碼，而是學會「如何跟 AI 溝通」。一開始我只說「夜市風」，結果太黑太暗；後來我學會描述細節，讓 AI 幫我調整成深灰色配金色，質感馬上提升。
    
    過程中最大的挑戰是「圖片跑不出來」，雖然 AI 給了很多建議，但最後我發現**自己動手修改程式碼**是最快也最精準的方法。這次作業讓我明白，AI 是很好的助手，但最後的把關者還是我自己。看著原本破圖的網頁變成現在這樣圖文並茂的樣子，非常有成就感！
